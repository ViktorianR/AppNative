управление последовательностью

это способ верного написания спец.символов в переменных.

Например:

в значении переменной нужно написать слово "can't";

конечно можно сделать и так: " can't "; - апостроф внутри двойных кавычек будет распознан как элемент строки, но в некоторых случаях

это может привести к ошибкам, потому как все данные при отправке на сервер переводятся в строки.

Для избежания ошибок используют ЭКРАНЫ " can\'t "; теперь символ стоящий после слеша будет распознан как элемент строки в любом случае

Если сравнивать строки, то буквы нижнего регистра будут "БОЛЬШЕ" чем буквы верхнего. Т.к буквы считаются большими если они стоят позже. toLowerCase('фраза') - приводит строку в нижний регистр. toUpperCase('фраза') - приводит строку в верхний регистр.

(2 >= 1) - два больше или равно одному

Это были НЕ строгие равенства
Строгие равенства сравнивают еще и типы данных

логиечские операторы

|| - логическое "или", используется в условных конструкциях ( 2 >= 3 || 2 <= 4 ) в этом случае выполняется первое условие

которое вернет true (низкий приоритет)

&& - логическое "и" так же используется в условных конструкциях но в отличии от "или" требует выполнения всех условий с которыми она связана

иначе возвращает false (высокий приоритет)

условные конструкции

это конструкции которые выполняют разные блоки кода в зависимости от выполнения условий

if()
{

//выполнится если true

}
else
{

//выполнится если false

}

else if() - альтернативная конструкция внутри if, для создания дополнительного условия.

(!name) - означает что переменная имеет значение null, undefined, 0, "" или NaN

alert( true || true ); // true
alert( false || true ); // true
alert( true || false ); // true
alert( false || false ); // false

alert( true && true ); // true
alert( false && true ); // false
alert( true && false ); // false
alert( false && false ); // false
конструкция switch - нужна для создания вариативности результата, используется строгое сравнение

switch(i)
{
case 2:

//Вывод если переменная i === 2;

break; // конец выполнения

case 3:

//Вывод если переменная i === 3;

break; // конец выполнения

default:

// Аналог else
break;
}
alert( null || 2 || undefined ); //выведет 2 потому как null == false, а алерт выведет первый true который найдет.

alert( alert(1) || 2 || alert(3) ); //выведет сначала первый алерт потом 2

alert( 1 && null && 2 );
//выведет null потому как это первое ложное значение.

alert( alert(1) && alert(2) );
//выведет 1 потом undefined, так как alert ничего не возвращает, и после первого вывода получит undefined

alert( null || 2 && 3 || 4 );
//выведет 3 потому как && выполняется первым по приоритету, а оно возвращает true с последним значением,

после чего подставляется в конструкцию "или" и становится первым истиным значением после null.
